# chess-T1 — Полный контекст для разработки ГИ

## 1. О проекте

Программа ГИ chess-T1 — настольное Windows-приложение (графический интерфейс) для игры chess-T1, частично схожей с шахматами. Это интерактивная доска с фигурами, перемещаемыми мышью. ИИ-противник отсутствует. Два игрока играют за одним ПК.

**Платформа:** Windows XP, 7, 10, 11 (нативное приложение).
**Язык интерфейса:** только русский.
**Образец:** программа «Кветка» (бесплатная шахматная доска без ИИ).
**Размер дистрибутива:** не более 10–15 МБ.

---

## 2. Игровое поле

- Доска 8×8, нотация: вертикали a–h, горизонтали 1–8.
- Стандартные клетки: чёрные (тёмно-серые) и белые.
- **Клетки замков** — выделены светло-серым со штриховкой (если штриховка затруднена — однородный светло-серый, заметно отличающийся от чёрных и белых).
  - Замок белых: c1, d1, e1, f1
  - Замок чёрных: c8, d8, e8, f8
- Обрамление: синий/голубой цвет (как в Кветке).
- Двойные линии между горизонталями 3–4, 5–6 и жирная линия между 4–5 — НЕ обязательны для 1-го уровня.
- Размер окна — максимально возможный для экрана пользователя.
- Корректное отображение при разных разрешениях.

## 3. Фигуры

### 3.1. Состав (7 видов, по 2 цвета = 14 иконок)

| Название     | Сокр. | Аналог в шахматах     | Тип фигуры       | Примечание                    |
|-------------|-------|----------------------|-------------------|-------------------------------|
| Король      | Кр    | Король               | Королевская       | —                             |
| Коннет      | Кт    | Ферзь (частично)     | Королевская       | Дальность ограничена          |
| Принц       | Пр    | Слон (частично)      | Королевская       | Может превращаться            |
| Риттер      | Рт    | Ладья (частично)     | Некоролевская     | Дальность 1–2 клетки          |
| Кнехт       | Кн    | Пешка                | Некоролевская     | Превращается в ВК             |
| Вер Кнехт   | ВК    | Пешка-ветеран        | Некоролевская     | Превращается далее            |
| Разведчик   | Рк    | Конь                 | Некоролевская     | Г-образный ход, спецход       |

**Королевские фигуры:** Король, Коннет, Принц — в реальных правилах могут входить в замки. Программа 1-го уровня это НЕ ограничивает.

### 3.2. Иконки фигур

- Формат: SVG (предоставляет заказчик).
- Высота всех иконок одинаковая, ширина различается.
- Соотношение стороны клетки к высоте иконки: **10:9**.
- Иконка автоматически центрируется в клетке по горизонтали. Нижний край — с небольшим отступом от нижнего края клетки.
- В каждой клетке может находиться **только одна** фигура.

### 3.3. Начальная расстановка

```
Горизонталь 8 (чёрные): a8-Рт  b8-Рк  c8-Пр  d8-Кт  e8-Кр  f8-Пр  g8-Рк  h8-Рт
Горизонталь 7 (чёрные): a7-Кн  b7-Кн  c7-Кн  d7-Кн  e7-Кн  f7-Кн  g7-Кн  h7-Кн
Горизонтали 6–3: пустые
Горизонталь 2 (белые):  a2-Кн  b2-Кн  c2-Кн  d2-Кн  e2-Кн  f2-Кн  g2-Кн  h2-Кн
Горизонталь 1 (белые):  a1-Рт  b1-Рк  c1-Пр  d1-Кт  e1-Кр  f1-Пр  g1-Рк  h1-Рт
```

## 4. Три состояния интерфейса

### 4.1. Стартовый Основной вид ГИ
- Когда: после запуска программы и закрытия Вводной страницы; после завершения сеанса партии.
- Ни Партия, ни Анализ не активны (кнопки не подсвечиваются).
- Строка индикации хода — **отсутствует**.
- На доске — начальная расстановка фигур.
- **Активные кнопки (верхний ряд):** Партия, Анализ, Свернуть, Поверх всех окон, Закрыть.
- **Активные кнопки (нижний ряд):** Меню, Изменить размер.
- **Заморожены:** Начальная расстановка, Предыдущий ход, Следующий ход, Реверс.
- **Меню — активны только:** О программе, Создать ярлык, Выход. Остальные заморожены.

### 4.2. Базовый Основной вид ГИ
- Когда: режим Партия (стадия Игра) или режим Анализ (стадия Игра).
- Есть **строка индикации** номера и очереди хода (справа от кнопки Меню).
- Кнопка активного режима (Партия или Анализ) подсвечивается.
- **Все кнопки активны** (кроме описанных исключений в таблицах поведения кнопок).
- **Меню — все пункты активны** (кроме заморожены на стадии «Задать позицию»).

### 4.3. Расширенный вид ГИ
- Когда: режим Анализ, стадия «Задать позицию».
- Слева от доски — касса белых фигур, справа — касса чёрных.
- Игровое поле пустое (без фигур).
- Добавляются кнопки: Сброс, Готово/Ok, 1-й ход, Удалить фигуру.
- **Активные:** все верхние + Меню, Реверс, Изменить размер, Сброс, Ok, 1-й ход, Удалить фигуру.
- **Заморожены:** Предыдущий ход, Следующий ход.
- **Меню — заморожены:** Сохранить позицию, Сохранить позицию как, Завершить партию.

## 5. Вводная страница (A00)

- При каждом запуске программы (по умолчанию) открывается окно с вводным текстом.
- Ширина = ширине Основного вида ГИ.
- **Элементы:**
  - Символ программы (не кнопка, декоративный)
  - Кнопка «Основной режим» — закрывает Вводную, открывает Основной вид ГИ
  - Кнопки: Свернуть, Поверх всех окон, Закрыть программу
  - Кнопка «Пропустить вводный текст / перейти в основной режим» — закрывает без запоминания
  - Кнопка **«Пропустить и не показывать больше»** — при последующих запусках Вводная не появляется
  - Окно с текстом + бегунок прокрутки
  - Кнопка «Изменить размер» (диагональная стрелка)
- Доступна через Меню → «О программе». После закрытия — возврат в тот же режим и позицию.
- **Исключение:** если программа закрылась без завершения партии, при следующем запуске Вводная НЕ показывается.

## 7. Ходы (режимы Партия и Анализ, стадия Игра)

### 7.1. Что программа реализует

1. **Автоматическая очерёдность ходов.** Начинают белые. После хода белых — только чёрные, и наоборот. В Анализе первый ход задаёт пользователь.

2. **Запрет хода на свою фигуру.** Нельзя поставить фигуру на клетку, занятую фигурой того же цвета. Программа блокирует.

3. **Взятие.** Ход на клетку с фигурой противника: фигура противника исчезает, атакующая занимает клетку.

4. **Превращения фигур** (см. раздел 11).

5. **Спецход Разведчика** (см. раздел 12).

### 7.2. Что программа НЕ реализует (игроки контролируют сами)

- Проверку правильности ходов по правилам подвижности фигур
- Силовую механику (перевес в силе при взятии)
- Ограничение подвижности королевских фигур в замке
- Определение шаха, мата, захвата замка
- Контроль Г-образного движения Разведчика

### 7.3. Строка индикации номера и очереди хода

Расположена справа от кнопки «Меню» в базовом Основном виде ГИ. Показывает номер **предстоящего** хода и чья очередь.

- **Ход белых:** `1. __ хб` (номер, точка, пробелы, «хб»)
- **Ход чёрных:** `8 … __ хч` (номер, пробел, три точки, пробелы, «хч»)

Нумерация по шахматной логике: номер увеличивается после хода чёрных.
Пример последовательности: `1. __ хб` → `1 … __ хч` → `2. __ хб` → `2 … __ хч` → `3. __ хб`

### 7.4. Техника хода (Drag & Drop)

1. Пользователь нажимает и удерживает ЛКМ на фигуре.
2. Перетаскивает на целевую клетку.
3. Отпускает ЛКМ.
4. Иконка автоматически центрируется в целевой клетке (как в Кветке).

### 7.5. Подсветка клеток

- При начале перемещения — подсвечивается **стартовая** клетка.
- При перетаскивании — подсвечивается клетка **под курсором**.
- После фиксации хода — целевая клетка **слабо подсвечена** (индикация последнего хода).

### 7.6. Поведение при выходе за пределы доски

- Отпускание фигуры **между клетками** — иконка «прилипает» к последней подсвеченной клетке (последней, через которую её пронесли).
- Вынос **за пределы доски** — иконка исчезает. На краю интерфейса появляется короткая горизонтальная двусторонняя стрелка.

### 7.7. История ходов

- Кнопки «Предыдущий ход» / «Следующий ход» прокручивают позиции по всей истории.
- Если пользователь откатился назад и сделал **новый ход** (отличный от прежнего) — старая ветка удаляется. Сохраняется только новая.
- Запись ходов (нотация) НЕ отображается.
- Сохранение/загрузка партии в файл НЕ требуется.

## 8. Режим «Партия»

### 8.1. Запуск

Нажатие «Партия» из стартового Основного вида ГИ:
1. Открывается диалоговое окно «Создать папку для партии».
2. Пользователь выбирает местоположение (по умолчанию — Рабочий стол, или через «Обзор»).
3. Вписывает название папки (название партии).
4. Можно закрыть окно крестиком (не создавать папку).
5. После создания/отказа — переход в базовый Основной вид ГИ с начальной позицией фигур и строкой индикации `1. __ хб`.

### 8.2. Поведение кнопки «Партия» в разных состояниях

| Текущее состояние | Поведение при нажатии |
|---|---|
| Стартовый вид | Запускает сеанс Партии (см. 8.1) |
| Уже в режиме Партия | Заморожена (но подсвечена) |
| Анализ, стадия «Задать позицию» | Заморожена |
| Анализ, стадия Игра | Завершает сеанс (без сохранения), переход в стартовый вид |

### 8.3. Поведение кнопки «Анализ» в разных состояниях

| Текущее состояние | Поведение при нажатии |
|---|---|
| Стартовый вид | Запускает режим Анализ (Расширенный вид) |
| Уже в режиме Анализ | Заморожена (но подсвечена) |
| Режим Партия | Завершает сеанс (без сохранения), переход в стартовый вид |

## 9. Режим «Анализ»

### 9.1. Стадия «Задать позицию» (Расширенный вид)

- Кассы фигур: слева — белые, справа — чёрные. Кассы **всегда полные** (можно брать любую фигуру неограниченно).
- Игровое поле: **пустое** (без фигур).
- Расстановка: Drag & Drop из кассы на поле. При захвате из кассы происходит **копирование** — оригинал остаётся в кассе.
- **Кнопка «Сброс»:** очищает поле от всех фигур.
- **Кнопка «Начальная расстановка»:** выставляет начальную позицию, остаёмся в режиме Анализ (стадия «Задать позицию»).
- **Кнопка «Удалить фигуру»:** ЛКМ на фигуру (клетка подсвечивается) → нажать кнопку → фигура удаляется.
- **Кнопка «1-й ход»:** два прямоугольника — белый и чёрный. Нажатие задаёт очерёдность первого хода. Выбор индицируется кружком. По умолчанию — белые.
- **Кнопка «Готово/Ok»:** фиксирует позицию, открывает окно «Создать папку для партии», переход в стадию Игра (базовый Основной вид). Строка индикации показывает 1-й номер хода и выбранную очерёдность.

### 9.2. Реверс на стадии «Задать позицию»

Доска поворачивается на 180°. Кассы меняются местами: левая (белые) ↔ правая (чёрные).

## 10. Меню, закрытие, сеансы

### 10.1. Меню (обновлённый состав — 6 пунктов)

| Пункт | Действие | Заморожен |
|---|---|---|
| О программе | Открывает Вводную страницу. После закрытия — возврат в текущий режим/позицию | — |
| Сохранить позицию | Сохраняет скриншот (PNG), имя = текст строки индикации. На ~1.5 сек появляется сообщение «Текущая позиция сохранена» | На стадии «Задать позицию»; в стартовом виде |
| Сохранить позицию как | Диалоговое окно, можно изменить имя файла | На стадии «Задать позицию»; в стартовом виде |
| Завершить партию | Диалог с кнопкой «Завершить партию» + галочка «Сохранить позицию» + крестик отмены. После завершения → стартовый вид | На стадии «Задать позицию»; в стартовом виде |
| Создать ярлык | Создаёт ярлык на рабочем столе. Если уже есть — не работает | — |
| Выход | Аналогично кнопке «Закрыть» | — |

**Расположение Меню:** раскрывается влево за пределы окна ГИ. Правый край прямоугольника — впритык к колонке цифр горизонталей. НЕ перекрывает игровое поле. Размер окна ГИ не меняется.

### 10.2. Закрытие программы (кнопка «Закрыть» / Меню → «Выход»)

**Если сеанс партии не завершён** — появляется диалоговое окно:
- **Кнопка «Завершить партию и закрыть программу»** + галочка «Сохранить позицию»
- **Кнопка «Закрыть программу — не завершая партию»** + галочка «Сохранить позицию»
- **Крестик** — отмена закрытия

**Если сеанс уже завершён** (или стартовый вид) — программа просто закрывается.

### 10.3. Сохранение скриншотов позиций

- **Формат:** PNG
- **Имя файла:** текст строки индикации (напр.: `8 … __ хч.png`)
- **Куда:** в папку партии (если создана), иначе — в стандартную папку Изображений Windows
- Если файл существует — предложение заменить или пропустить

### 10.4. Сеанс партии — логика запуска и перезапуска

**Закрытие БЕЗ завершения партии:**
- При следующем запуске — сразу (без Вводной страницы) открывается базовый Основной вид ГИ в режиме Партия.
- Позиция на доске — в точности та, что была при закрытии.
- Строка индикации — тот же номер и очерёдность хода.
- Историю ходов сохранять необязательно.

**Закрытие С завершением партии:**
- Следующий запуск — штатный режим: Вводная страница (или стартовый вид, если пользователь ранее нажал «Пропустить и не показывать больше»).

**При последующих запусках** (если сеанс был завершён):
- Открывается стартовый Основной вид ГИ.
- Пользователь выбирает Партия или Анализ.

## 11. Превращения фигур

### 11.1. Кнехт → Вер Кнехт (АВТОМАТИЧЕСКОЕ)

| Условие | Результат |
|---|---|
| Белый Кнехт попадает на **любую клетку 6-й горизонтали** | Мгновенно заменяется на белого Вер Кнехта |
| Чёрный Кнехт попадает на **любую клетку 3-й горизонтали** | Мгновенно заменяется на чёрного Вер Кнехта |

Никакого окна выбора. Превращение мгновенное. Включая ходы со взятием.

### 11.2. Вер Кнехт → выбор из 4 фигур (крайние клетки)

| Условие | Окно выбора | Расположение окна |
|---|---|---|
| Белый ВК на **a8, b8, g8, h8** | 4 иконки белых: Кт, Пр, Рт, Рк | Правый верхний угол |
| Чёрный ВК на **a1, b1, g1, h1** | 4 иконки чёрных: Кт, Пр, Рт, Рк | Правый нижний угол |

### 11.3. Вер Кнехт → выбор из 2 фигур (клетки замка противника)

| Условие | Окно выбора | Расположение окна |
|---|---|---|
| Белый ВК на **c8, d8, e8, f8** | 2 иконки белых: Кт, Пр | Правый верхний угол |
| Чёрный ВК на **c1, d1, e1, f1** | 2 иконки чёрных: Кт, Пр | Правый нижний угол |

### 11.4. Принц → выбор из 2 фигур (клетки замка противника)

| Условие | Окно выбора | Расположение окна |
|---|---|---|
| Белый Пр на **c8, d8, e8, f8** | 2 иконки белых: Кт, Пр | Правый верхний угол |
| Чёрный Пр на **c1, d1, e1, f1** | 2 иконки чёрных: Кт, Пр | Правый нижний угол |

### 11.5. Общие правила окон превращения

- Иконки в окне выбора — **~80% от размера** иконок на доске.
- Клетка превращения **подсвечивается** до завершения выбора.
- Кнопка «Свернуть» **замораживается** до завершения превращения.
- После выбора: фигура заменяется, окно исчезает, клетка остаётся **слабо подсвеченной** (индикация последнего хода).
- Пользователь выбирает фигуру нажатием ЛКМ на иконку в окне.

## 12. Спецход Разведчика — «взятие с разменом»

### Условия:
- Фигура **противника** стоит на клетке **любого замка**: c1, d1, e1, f1 (замок белых) или c8, d8, e8, f8 (замок чёрных).
- Разведчик (любого цвета) делает ход на эту клетку.

### Результат:
- **Обе фигуры исчезают** с доски: и Разведчик, и фигура противника.
- Очередь хода переходит к противоположной стороне.
- Клетка остаётся **слабо подсвеченной** (индикация последнего хода).

### Реализация:
Программа должна определить: если фигура, делающая ход на клетку замка — это Разведчик, И на клетке стоит фигура противника → убрать обе фигуры.

**Альтернативный вариант** (если автоматический затруднён): реализация через кнопку «Удалить фигуру» (по согласованию с заказчиком). В этом случае кнопка «Удалить фигуру» присутствует в Основном виде ГИ.

## 14. Приёмка работы

- Сдача единовременная (не поэтапная).
- Перед сдачей — согласование дизайна, размеров, цветовых решений.
- Приёмка тестированием: проверка всех режимов, кнопок, ходов, превращений, спецхода Разведчика, сохранения скриншотов, восстановления позиции после перезапуска.

## 15. Схема переходов между состояниями (state machine)

```
ЗАПУСК ПРОГРАММЫ
    │
    ├── (сеанс НЕ завершён при прошлом закрытии)
    │       └── Базовый вид (Партия) с сохранённой позицией
    │
    └── (сеанс был завершён / первый запуск)
            │
            ├── (кнопка «Не показывать больше» НЕ была нажата)
            │       └── Вводная страница → Стартовый вид
            │
            └── (кнопка «Не показывать больше» БЫЛА нажата)
                    └── Стартовый вид

СТАРТОВЫЙ ВИД
    ├── [Партия] → Окно «Создать папку» → Базовый вид (Партия), 1. __ хб
    ├── [Анализ] → Расширенный вид (стадия «Задать позицию»)
    ├── [Закрыть] → Выход
    └── [Меню] → О программе / Создать ярлык / Выход

РАСШИРЕННЫЙ ВИД (Анализ, стадия «Задать позицию»)
    ├── [Ok] → Окно «Создать папку» → Базовый вид (Анализ, стадия Игра)
    ├── [Начальная расстановка] → остаёмся в Расширенном виде с начальной позицией
    ├── [Партия] → заморожена
    └── [Анализ] → заморожена (подсвечена)

БАЗОВЫЙ ВИД (Партия, стадия Игра)
    ├── [Партия] → заморожена (подсвечена)
    ├── [Анализ] → завершает сеанс → Стартовый вид
    ├── [Начальная расстановка] → завершает сеанс → Стартовый вид
    ├── [Меню → Завершить партию] → Диалог → Стартовый вид
    └── [Закрыть] → Диалог закрытия

БАЗОВЫЙ ВИД (Анализ, стадия Игра)
    ├── [Анализ] → заморожена (подсвечена)
    ├── [Партия] → завершает сеанс → Стартовый вид
    ├── [Начальная расстановка] → завершает сеанс → Стартовый вид
    ├── [Меню → Завершить партию] → Диалог → Стартовый вид
    └── [Закрыть] → Диалог закрытия
```
